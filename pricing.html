Frequently Asked Questions

Q:What is a concurrent build?

Answer

Concurrent builds states number of builds that can run at the same time,
Concurrent builds allow you to run multiple builds at same. Every commit triggers a one job by default. If two commits have been pushed on the Free plan, they'll be run right away and ubsequent commits have to wait until the builds finished.

Q: What are parallel test pipelines?

Answer

Parallel test pipelines allow you to split up your build into multiple pipelines to speed up your testing.
